BROWSER REQUIREMENTS AND CONFIGURATIONS

To ensure consistency and the best user experience in web browser when accessing Vocareum Lab, we would recommend students to use Google Chrome (Click the link to install if it is not available on your computer). As best practice to configure the browser, please follow the configurations below.

  1. Update the web browser to the latest version.

  2. System Requirements (Please refer here for the latest information):

    1. Windows

      1. Windows 7, Windows 8, Windows 8.1 or Windows 10

      2. An Intel Pentium 4 processor or later that’s SSE3 capable

    2. MAC OS

      1. OS X EI Capitan 10.11 or later

    3. Linux OS

      1. 64 bit Ubuntu 18.04+, Debian 10+, openSUSE 15.2+, or Fedora Linux 32+

      2. An Intel Pentium 4 processor or later that’s SSE3 capable

  3. Configure browser to allow third-party cookies from Vocareum. The screenshots provided at the time of writing is based on version 102.0.5005.63.

    1. Open Chrome browser.

    2. On the URL bar, enter chrome://settings/cookies

  1. Scroll down the page until you see this section – Sites that can always use cookies and click on the Add button.

  1. In the Add a site box, enter [.]labs.vocareum.com* into the Site prompt and check Including third-party cookies on this site.

  1. Click Add button to add and the prompt box will close.

  1. You will notice that your entry is now displayed at this section.

  1. Restart your Chrome browser.

    1. Go to https://canvas.suss.edu.sg and log in to your account.

  1. Navigate to your course site and select Vocareum Labs under the Courses menu.

  1. For first-time users, you will encounter the Vocareum Terms and Conditions, please scroll down to indicate that you agree with the terms and conditions.

  1. Subsequently, the platform will automatically show the lab that is relevant to your course. To access the lab environment, please click on My Work button.

3.2 CREATING LAUNCH.JSON FILE FOR DEBUGGING FLASK APPLICATIONS (if applicable to your course)

Note: This section assumes that you have already opened your folder at the application root level where it contains the app.py file.

  1. In the IDE left panel, click on the Run and Debug button that is highlighted in orange.

  1. Under the RUN AND DEBUG section, please click on create a launch.json file link. If there is an existing Python: Flask configuration, please skip step c-g.

  1. The IDE will prompt for the environment. Please select Python.

  1. Under the debug configuration prompt, click Flask.

  1. The IDE will automatically open the launch.json configuration and the necessary contents will be populated.

  1. Please add the following changes under the env section in the launch.json file.

    1. Please add a comma at the end of the last entry behind the word development

    2. Please add a new line “FLASK_RUN_HOST”: “0.0.0.0”

  1. Please save the launch.json file after making the changes.

  2. You may now run your flask application in debugging mode by clicking on the green play button.

APPENDIX A-1 [DOWNLOADING FILE FROM VIRTUAL DESKTOP]

To download file from virtual desktop to your local computer, please use the steps below:

  1. Copy the file(s) that you need.

  2. Click to open File Manager from Applications menu.

  1. Navigate to the following path in File Manager:

/home/labsuser/

  1. Paste those file(s) into this folder. If there are multiple files, you may wish to create a temporary folder to hold it first.

  1. Navigate to WorkSpace Terminal and click on Reload icon as pointed to by arrow below. This action will reload the contents of the work folder.

  1. To ensure the file(s) is reloaded correctly, expand work/Temp folder to verify that those file(s) is present.

  1. Select the file(s) that you need and then click Download button.

  1. System will compress the selected file into Zip. Click Download zipped source link to download and save those file(s) into local computer.

APPENDIX A-2 [UPLOADING OF FILE TO VIRTUAL DESKTOP]

To upload file from your local computer to virtual desktop, please use the steps below:

  1. Go to WorkSpace Terminal and select work folder.

  1. Click on Upload button and open the file that you need.

  1. System will start to process the upload and will prompt successful when completed.

  1. Navigate to Virtual Desktop to verify those uploaded files. They should be located in the File Manager when you access this path.

/home/labsuser/

    2. Then click Start Lab to start a new lab session.
